My Understanding of HTML Structure and Basic Tags

HTML uses a "boxes inside boxes" structure to build webpage skeleton.

1. The HTML Structure

  • DOCTYPE html:
    This tells the browser this is a modern HTML page. note it has an exclamation mark and no closing tag.
  • html tag:
    This is the very outer container.
    Every single piece of code you write must sit inside the opening and the closing tags.
    If it's not inside this box, it doesn't exist to the browser.
  • head tag:
    This container holds information about the page that user doesn't see on the main screen.
    It holds things like character settings (charset="UTF-8") and the title tag.
    The text between the title opening and closing tags is what that appears at the very top of your screen.
  • body tag:
    This is the main container
    This holds everything you want to actually show up on the screen.
    Headings, paragraphs, images, and links.

2. Purpose of Basic Tags & Attributes

Tags act as containers, tell the browser how to treat the text they surround.
Tags usually come in pairs, Only Opening tags (1st tag) can hold attributes to provide extra details about the content.
Every attribute must have a name and a value

  • Heading (h1):
    Acts as the main "Book Title" for the page.
    Google looks at this tag first to rank and understand what the webpage is about.
    which is why it should only be used once per page.
  • Paragraphs (p):
    It groups sentences into blocks and adds vertical spaces so the text doesn't smash together.
  • Tag Pairs:
    Act like an ON/OFF switch.
    The opening tag turns a style ON, and the closing tag (/) turns it OFF so the browser knows exactly where to stop.
  • Attributes:
    Written inside opening tags e.g; (name="value") to provide extra details about the content between the tags
